# Teamagent プロジェクト

## 概要
Claude Code Agent Teams を使った並列開発プロジェクト。
チームエージェントの役割は実行時に動的に指定する運用方式を採用。

## チームエージェント設定
- Agent Teams 機能: 有効 (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`)
- 表示モード: `in-process`（Windows環境）
- tmux-windows: インストール済み

---

## チーム運用ガイド

### チームの起動方法
Claude Code を起動し、タスクに応じて役割テンプレートを選んでチーム作成を依頼する。

### 役割テンプレート

タスクの種類に応じて以下のテンプレートを参考にチームを構成する。
役割名・人数はタスクに合わせて自由にカスタマイズ可能。

#### 開発タスク向け（3人）
```
エージェントチームを作成して並列で作業してください：
- 設計者: アーキテクチャ設計・技術選定を担当
- 実装者: コーディング・ファイル生成を担当
- 品質担当: テスト・コードレビュー・チェックリスト検証を担当
```

#### 調査・リサーチ向け（3人）
```
エージェントチームを作成して並列で調査してください：
- リサーチャーA: [調査テーマ1]を担当
- リサーチャーB: [調査テーマ2]を担当
- アナリスト: 両者の調査結果を統合・分析して最終レポートを作成
```

#### デバッグ・問題解決向け（3〜5人）
```
エージェントチームを作成して仮説ごとに並列調査してください：
- 調査員1: [仮説1]を検証
- 調査員2: [仮説2]を検証
- 調査員3: [仮説3]を検証
互いの仮説に反証を試み、最も有力な原因を特定すること。
```

#### スキル作成向け（3人）
```
エージェントチームを作成してスキルを並列で構築してください：
- 設計者: ユースケース分析・ワークフロー設計を担当
- 実装者: SKILL.md・referencesファイルの生成を担当
- テスター: チェックリスト検証・テスト計画を担当
```

#### PR・コードレビュー向け（3人）
```
エージェントチームを作成してPRをレビューしてください：
- セキュリティ担当: セキュリティの観点でレビュー
- パフォーマンス担当: パフォーマンス・効率の観点でレビュー
- テスト担当: テストカバレッジ・品質の観点でレビュー
```

### カスタムチームの作り方
テンプレートに合わない場合は自由に役割を指定できる:
```
エージェントチームを作成してください。
チームメイト:
- [役割名]: [担当内容の説明]
- [役割名]: [担当内容の説明]
各チームメイトは[モデル名: Sonnet / Opus]を使用。
実装前にプラン承認を必須とする。
```

## チームメイト操作

| キー | 操作 |
|------|------|
| `Shift+Down` | チームメイト間を移動 |
| `Enter` | チームメイトのセッションを表示 |
| `Escape` | 現在のターンを中断 |
| `Ctrl+T` | タスクリストの表示/非表示 |

## 運用ルール

### チーム構成
- チームメイトは **3〜5人** が最適（コスト・効率のバランス）
- 1チームメイトあたり **5〜6タスク** が理想
- 同じファイルを複数のチームメイトが編集しないよう分担する

### チーム終了時
1. 全チームメイトの作業完了を確認
2. リーダーに「チームをクリーンアップして」と指示
3. チームメイトが残っていたら先に「[名前]チームメイトをシャットダウンして」

### 注意事項
- Windows環境ではスプリットペインモード（tmux分割表示）に制限あり
- in-processモードを使用
- チームメイトはリーダーの権限設定を引き継ぐ
- 各チームメイトは独立したコンテキストウィンドウを持つ（リーダーの会話履歴は引き継がない）
- トークン消費はチームメイト数に比例して増加する

## スキル管理

### スキル一覧の確認
`.claude/skills/README.md` にスキル一覧表がある。新規作成・更新・削除時は必ず一覧を更新する。

### スキル新規作成
1. 「新しいスキルを作りたい」または「skill-creator-maxを使って」と入力
2. 一問一答の対話に従いスキルを設計・生成
3. `.claude/skills/README.md` の一覧表に追記
4. git commit & push

### スキル更新
1. 該当スキルの `SKILL.md` や `references/` を編集
2. `metadata.version` を更新（パッチ: 1.0.1 / マイナー: 1.1.0 / メジャー: 2.0.0）
3. `.claude/skills/README.md` のバージョンも更新

### スキル削除
1. `.claude/skills/[name]/` ディレクトリを削除
2. `.claude/skills/README.md` から該当行を削除
3. git commit & push

### スキルの品質基準
新規スキル作成後は以下を確認する:
- YAML frontmatter の `name` がフォルダ名と一致（kebab-case）
- `description` にトリガーワードが3個以上
- ワークフローが Step 番号で完結している
- エラーハンドリングセクションがある
- ファイル保存スキルの場合は Write ツールが CRITICAL で明示されている
