# 01 — Excelシート構成・カラム定義

## 共通スタイルルール

### ヘッダー行
- 背景色: 濃い青（#1F4E79）
- 文字色: 白
- フォント: 太字、サイズ11
- 配置: 中央揃え
- フィルター: ヘッダー行にオートフィルターを設定

### データ行
- フォント: サイズ10
- 配置: 左揃え（テキスト）、中央揃え（ステータス・日付）
- 折り返し: 長文セルは折り返し表示を有効にする
- 罫線: 全セルに細い罫線を設定

### 列幅の目安
- No.: 6
- 短いテキスト（ステータス・優先度等）: 12〜15
- サービス名・タスク名: 25〜30
- 説明・備考: 40〜50
- URL: 60

---

## 管理用資料（aws-eol-management-YYYY-MM-DD.xlsx）

### Sheet1: EOL対象サービス一覧

全サービスを横断で一覧化し、対応状況を俯瞰する。

| カラム名 | 幅 | 説明 |
|---------|-----|------|
| No. | 6 | 連番 |
| AWSアカウント名 | 25 | 対象アカウントの識別名 |
| AWSアカウントID | 18 | 12桁のアカウントID（判明している場合） |
| サービス名 | 25 | AWSサービス名（例: RDS MySQL） |
| リソース名/ID | 30 | 具体的なリソース識別子（判明している場合） |
| 現行バージョン | 15 | 現在使用中のバージョン |
| EOL日程 | 15 | AWS公式のEOL/EOS日付（YYYY-MM-DD） |
| 延長サポート終了日 | 18 | 延長サポートがある場合の終了日 |
| 移行先 | 25 | AWS推奨の移行先バージョン/サービス |
| 優先度 | 10 | 高/中/低 |
| ステータス | 12 | 未着手/調査中/対応中/完了 |
| 備考 | 40 | 補足事項 |

#### 優先度の色分け（条件付き書式）
- 高: 赤背景（#FFC7CE）
- 中: 黄背景（#FFEB9C）
- 低: 緑背景（#C6EFCE）

#### ステータスの色分け（条件付き書式）
- 未着手: 赤文字
- 調査中: 青文字
- 対応中: オレンジ文字
- 完了: 緑文字

---

### Sheet2: 対応タスク一覧

各サービスのEOL対応に必要なタスクを管理する。

| カラム名 | 幅 | 説明 |
|---------|-----|------|
| No. | 6 | 連番 |
| 対象サービス | 25 | 関連するAWSサービス名 |
| タスク名 | 35 | 具体的な作業内容 |
| 担当者 | 15 | 担当者名（空欄で出力、後から記入） |
| 開始予定日 | 15 | 作業開始予定日（YYYY-MM-DD） |
| 期限 | 15 | 作業完了期限（YYYY-MM-DD） |
| ステータス | 12 | 未着手/進行中/完了/保留 |
| 依存関係 | 30 | 先行タスクの番号（例: No.3完了後） |
| 備考 | 40 | 補足事項 |

#### 標準タスクテンプレート（サービスごとに以下を生成）
1. EOL情報の詳細調査・確認
2. 影響範囲の洗い出し
3. 移行計画の策定
4. 検証環境での移行テスト
5. 本番環境での移行実施
6. 移行後の動作確認
7. 旧リソースのクリーンアップ

---

### Sheet3: スケジュール

EOL期限に基づくタイムラインを一覧化する。

| カラム名 | 幅 | 説明 |
|---------|-----|------|
| サービス名 | 25 | 対象サービス |
| EOL期限 | 15 | AWS公式のEOL日付 |
| 残日数 | 10 | 本日からEOL期限までの日数 |
| 調査完了目標 | 15 | 推奨: EOL 3ヶ月前 |
| 検証開始目標 | 15 | 推奨: EOL 2ヶ月前 |
| 本番対応目標 | 15 | 推奨: EOL 1ヶ月前 |
| バッファ期間 | 12 | EOL期限までの余裕日数 |
| 備考 | 40 | 延長サポート有無など |

#### 残日数の色分け（条件付き書式）
- 90日以上: 緑背景
- 30〜89日: 黄背景
- 30日未満: 赤背景

---

## 個別サービス資料（[サービス名]-eol-report-YYYY-MM-DD.xlsx）

### Sheet1: 調査内容まとめ

サービスの概要・影響範囲をキーバリュー形式で整理する。

| 項目（A列） | 内容（B列） |
|------------|------------|
| サービス名 | [正式サービス名] |
| 現行バージョン | [使用中のバージョン] |
| EOL対象バージョン/機能 | [EOLとなるバージョンや機能] |
| AWS公式アナウンス日 | [アナウンスが行われた日付] |
| EOL/EOS日程 | [公式のEOL日付] |
| 延長サポート | [あり/なし、ある場合は期間と費用] |
| 影響範囲（概要） | [EOLによる影響の概要説明] |
| セキュリティへの影響 | [パッチ停止等のセキュリティリスク] |
| 代替サービス/移行先 | [AWS推奨の移行先] |
| 移行の複雑度 | [高/中/低 と理由] |
| 推定ダウンタイム | [移行時のダウンタイム見込み] |
| 追加コスト | [移行に伴うコスト変動の見込み] |

- A列幅: 25、B列幅: 60
- A列: 太字、背景色 薄い青（#D6E4F0）
- B列: 折り返し表示

---

### Sheet2: EOL手順

移行に必要な作業手順をステップ形式で記述する。

| カラム名 | 幅 | 説明 |
|---------|-----|------|
| ステップNo. | 10 | 連番 |
| フェーズ | 15 | 準備/検証/実施/確認 |
| 作業内容 | 35 | 作業のタイトル |
| 詳細説明 | 50 | 具体的な手順の説明 |
| 前提条件 | 30 | この作業の前に必要なこと |
| 注意事項 | 30 | リスク・注意点 |
| 推定工数 | 12 | 作業時間の目安 |

---

### Sheet3: EOL期限

EOLに関する日程情報を整理する。

| 項目（A列） | 日付（B列） | 備考（C列） |
|------------|------------|------------|
| AWS公式EOL日 | YYYY-MM-DD | 公式発表の日付 |
| 延長サポート終了日 | YYYY-MM-DD | 延長サポートがある場合 |
| セキュリティパッチ提供終了日 | YYYY-MM-DD | パッチ停止日 |
| 推奨対応開始日 | YYYY-MM-DD | EOL 3ヶ月前を目安 |
| 検証環境テスト期限 | YYYY-MM-DD | EOL 2ヶ月前を目安 |
| 本番環境対応期限 | YYYY-MM-DD | EOL 1ヶ月前を目安 |
| 最終対応期限 | YYYY-MM-DD | これ以降はリスク許容 |

- A列幅: 30、B列幅: 18、C列幅: 40
- A列: 太字、背景色 薄い青（#D6E4F0）
- 期限を過ぎた日付: 赤文字（条件付き書式）

---

### Sheet4: 参考文献

調査に使用した公式ドキュメント・記事のリンク集。

| カラム名 | 幅 | 説明 |
|---------|-----|------|
| No. | 6 | 連番 |
| タイトル | 40 | ドキュメント・記事のタイトル |
| URL | 60 | リンク（ハイパーリンクとして設定） |
| 種別 | 18 | AWS公式ドキュメント/AWSブログ/サードパーティ |
| 概要 | 50 | 内容の簡単な説明 |

**CRITICAL: URLはExcelのハイパーリンクとして設定すること（クリックで開けるようにする）。**

openpyxlでのハイパーリンク設定方法:
```python
from openpyxl.utils import get_column_letter

cell = ws.cell(row=row_num, column=url_col, value=url)
cell.hyperlink = url
cell.style = 'Hyperlink'
```
